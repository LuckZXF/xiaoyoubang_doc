##功能模块

###1.用户模块：
####用户数据库表
```
字段：
用户id，用户账号（云信账号）,用户名，用户电话，用户昵称，用户密码（如与云信token分离则多一个字段记录云信token），性别，头像（存入云信），注册时间，毕业时间，学号，信用积分，财富积分，备注，登陆状态（是否在线），身份（是否为管理员，管理员操作管理系统）。
```
####特殊需要：
```
1.  注册接口，同时注册云信账号

2.  注册手机验证码（若没可用我的接口）

3.  登陆接口和登出接口（根据是否禁用登陆，更新登陆状态）

4.  修改个人资料接口

5.  查看他人财富分数和信用分数接口

6.  财富积分榜接口 返回财富分数前10的用户信息

7.  用户id信息获取接口 通过APP端传的id 返回用户信息所有，APP端根据需要展示字段内容

8.  在线人数接口（管理系统接口）

9.  管理员获取用户接口 返回前10个用户（APP端传参数0表示第一次加载或刷新）

10. 管理员获取更多用户 一次加载多10个用户（APP端传参数1.2.3...表示第2.3.4...次加载）

11. 管理员获取注册人数接口 （身份非管理员的所有用户数量）
```



###2.发帖模块：
####帖子详情数据库表：

```
字段：
帖子id，发帖用户id, 帖子标题， 帖子内容，帖子创建时间，借款信息id（帖子中的借款相关信息保存借款信息中），是否属于子回复（发帖和回复保存在同一张表里不同状态码决定是发帖还是回复），帖子审核（如果为借贷贴由管理系统决定是否通过，帖子状态为N，通过为Y，只返回Y状态的帖子给APP），点赞（帖子和回复具有点赞功能），帖子评论数量（父贴id存在一个回复数量标记），借贷贴借贷用途方向（1.旅游2.创业3.租房4.学习5.其他），借贷金额范畴（1：500元以下 2：500元至2000元 3：2000元至5000元 4：5000元至10000元 5：1W元以上），借贷期限范畴（1：1个月内 2：1一个月至3个月 3：3个月至半年 4：半年至一年）
父贴id（如果是回复 存在一个回复的帖子id），
```

####特殊需要：
```
1. 发帖的时候用户 借款帖子会填写借款的相关信息保存在另一张借款信息表中，同时根据金额和期限判断所在借贷范畴。
2. 评论接口，用户提交评论保存到帖子表里并附上父贴id
3. 评论点赞接口，提交点赞对应评论id，该回复点赞数量加一
4. 帖子获取接口，返回前10条帖子（APP端传参数0表示第一次加载或刷新，只返回已经通过审核的帖子）
5. 帖子加载接口，一次加载多10条帖子（APP端传参数1.2.3...表示第2.3.4...次加载）
6. 帖子评论获取接口，根据父贴id返回对应的所有回复
7. 热门帖子获取接口，返回所有帖子中评论数量最多的父贴信息
8. 发借贷帖子的手机验证接口 （App自行判断是否需要手机验证，验证通过再调发帖接口）
9. 特定帖子的获取， 根据 借贷用途||借贷金额范畴||借贷期限范畴（任意数量的筛选条件） 返回特定的帖子信息 刷新加载           参考4 5 （该接口需只返回审核通过的帖子）
10. 管理员获取未审核帖子（返回未审核帖子，获取刷新加载形式参照 4 5）
11. 帖子删除 （删除条件：身份为管理员或发帖用户id为当前id）

```


###3.借款模块：
####借款信息：

```
字段：借贷单id，借贷人id，出资人id，借贷单发起时间，借贷单金额，借贷周期，借贷单支付成功时间，借贷单还款时间，借贷单状态（审核，等待借贷，借贷中，还款成功）
```


####特殊需要：
```
1.借贷单获取接口 根据借贷人id 获取名下所有借贷单
2.借贷单支付接口 根据APP传的借贷单id 更改借贷单状态为借贷中，同时增加赞助人id财富分数
3.借贷单还款接口 根据APP传的借贷单id 更改借贷单状态为已还款，同事增加借贷人id信用分数
4.管理员获取借贷总金额接口  将所有状态为借贷中和还款成功的借贷单金额累加

```


###4.商品模块：
####商品信息数据库表

```
字段：
商品id，商品名称，商品数量，商品积分价格，商品说明，商品图片url，商品状态（商城归属或个人归属），用户id（若有则说明是用户已兑换的商品），兑换时间（若为个人归属）

```

####特殊需要：
```
1.商品兑换接口 根据前端传的商品id和数量更新当前用户的个人归属

2.个人已兑换商品信息接口 根据用户id返回已兑换的商品信息

```
